Issue Type: Security report
Recommended Visibility: Private vulnerability report
Suggested Title: Require `LW_API_20_BOT_SECRET` or an explicit dev override before accepting webhook callbacks

Description
The chat connector currently fails open when `LW_API_20_BOT_SECRET` is unset. `LineWorksClient.verify_signature()` returns `True` in that case, so a publicly deployed `/callback` endpoint can accept unsigned requests. Because the setup guide explicitly deploys this service to a public URL, an accidental env-var omission weakens the trust boundary from "signed webhook only" to "anyone who can POST JSON".

Component
- [x] Chat Connector
- [ ] Flowise Flow
- [ ] Demo MCP Server
- [ ] Documentation
- [ ] Other

Steps to Reproduce
1. Deploy `chat-connector` without setting `LW_API_20_BOT_SECRET`.
2. Send a crafted POST request to `/callback` with a plausible LINE WORKS message body and no valid `X-WORKS-Signature`.
3. Observe that `verify_signature()` returns `True` and the request continues through normal processing.

Expected Behavior
Public webhook handling should fail closed by default. If signature verification is intentionally disabled for local development, that should require an explicit opt-in such as `ALLOW_UNSIGNED_WEBHOOKS=true`.

Actual Behavior
Unsigned requests are accepted whenever `LW_API_20_BOT_SECRET` is missing.
