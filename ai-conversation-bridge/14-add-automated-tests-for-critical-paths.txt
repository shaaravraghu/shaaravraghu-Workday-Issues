Suggested Title: Add automated tests for webhook handling, response validation, and demo MCP tool behavior

## Problem
The repo includes `pytest` in `chat-connector/requirements.txt`, but there are no test files for the critical paths. That leaves several subtle regressions unguarded, including signature verification behavior, oversize-message handling, response validation semantics, and demo MCP tool consistency.

## Proposed Solution
Add a basic automated test suite that covers:
- webhook signature pass/fail behavior
- missing-config behavior in the callback route
- `ResponseValidator` fallback/hedging handling
- `request_my_time_off` semantics
- schema consistency for demo MCP tools such as `get_direct_reports`
