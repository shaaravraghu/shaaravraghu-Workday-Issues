Suggested Title: Make demo time-off request IDs unique across repeated submissions

## Description
`request_my_time_off()` builds request IDs using only `CURRENT_USER_WORKER_ID` and `start_date`. If the same worker submits multiple requests with the same start date, the generated IDs collide.

## Component
- [ ] Chat Connector
- [ ] Flowise Flow
- [x] Demo MCP Server
- [ ] Documentation
- [ ] Other

## Steps to Reproduce
1. Call `request_my_time_off(start_date="2026-06-01", end_date="2026-06-02", time_off_type="vacation")`.
2. Call it again with the same `start_date` for the same current worker.
3. Observe that both responses return the same `request_id`.

## Expected Behavior
Each submitted request should have a unique identifier.

## Actual Behavior
The request ID format is `TOR-{CURRENT_USER_WORKER_ID}-{start_date}`, which collides for repeated submissions.
