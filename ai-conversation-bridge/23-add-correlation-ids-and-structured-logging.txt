Issue Type: Feature Request
Recommended Visibility: Public issue
Suggested Title: Add correlation IDs and structured logging across the chat connector request path

## Problem
When a user reports that the bot did not respond, it is hard to trace a single request across webhook receipt, AI provider call, and LINE WORKS reply delivery. The enterprise hardening guide already recommends correlation IDs and structured logging, but the current implementation still uses plain text logs without a request identifier.

## Proposed Solution
Generate a request ID at `/callback` entry, attach it to all log records, and propagate it through outbound provider calls and LINE WORKS delivery calls.

- [x] Chat Connector (new platform, adapter improvement)

## Alternatives Considered

- Add external tracing later without updating application logs
