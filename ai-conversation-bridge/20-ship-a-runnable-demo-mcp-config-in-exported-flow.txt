Issue Type: Bug Report
Recommended Visibility: Public issue
Suggested Title: Ship a runnable demo MCP configuration in the exported Flowise template

## Description
The exported Flowise template includes a placeholder MCP URL (`https://YOUR_MCP_SERVER_URL/mcp`) and a bearer-token header, even though the demo MCP server in this repo has no authentication. This means the imported template is not runnable against the repo’s own demo server until the user manually edits the MCP configuration.

## Component
- [ ] Chat Connector
- [x] Flowise Flow
- [ ] Demo MCP Server
- [x] Documentation
- [ ] Other

## Steps to Reproduce
1. Import `flowise/flows/workday-mcp-agent.json` into Flowise.
2. Inspect the Custom MCP tool config inside the agent node.
3. Observe that the stored JSON points to a placeholder URL and includes an `Authorization` header.

## Expected Behavior
The default exported demo flow should either:
- already target a documented demo-safe config shape, or
- use an obviously disabled placeholder that does not imply auth for the unauthenticated demo server.

## Actual Behavior
The exported JSON bakes in a placeholder server URL and auth header that do not match the demo MCP server defaults.
