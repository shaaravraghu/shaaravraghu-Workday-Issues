Suggested Title: Return explicit timezone/offset information from `get_today_date_and_day_of_week`

## Description
`get_today_date_and_day_of_week()` returns a naive local date/time and hardcodes `"timezone": "server_local"`. In a cross-region APJ-oriented demo, that is ambiguous and makes it difficult for flows to reason about exact dates, offsets, and locale-sensitive time-off requests.

## Component
- [ ] Chat Connector
- [ ] Flowise Flow
- [x] Demo MCP Server
- [ ] Documentation
- [ ] Other

## Steps to Reproduce
1. Call `get_today_date_and_day_of_week()`.
2. Inspect the returned payload.
3. Observe that the response includes local clock time but not an actual timezone name or UTC offset.

## Expected Behavior
The tool should return explicit machine-readable time context, such as:
- ISO 8601 datetime
- UTC offset
- IANA timezone when available

## Actual Behavior
The response uses `datetime.now()` and the placeholder string `"server_local"`.
