Issue Type: Bug Report
Recommended Visibility: Public issue
Suggested Title: Validate time-off date ranges and entitlements in `request_my_time_off`

## Description
`request_my_time_off()` only validates the leave type. It accepts invalid date strings, end dates before start dates, and requests that may exceed the worker’s balance or eligibility rules.

## Component
- [ ] Chat Connector
- [ ] Flowise Flow
- [x] Demo MCP Server
- [ ] Documentation
- [ ] Other

## Steps to Reproduce
1. Call `request_my_time_off(start_date="not-a-date", end_date="2026-05-01", time_off_type="vacation")`.
2. Call `request_my_time_off(start_date="2026-05-10", end_date="2026-05-01", time_off_type="vacation")`.
3. Call `request_my_time_off()` for a leave type or duration that exceeds the current worker’s balances/eligibility.
4. Observe that the tool still returns `"status": "submitted"` in all of these cases.

## Expected Behavior
The demo should validate:
- date parsing
- start/end ordering
- eligibility for the requested leave type
- sufficient available balance when that information exists in mock data

## Actual Behavior
Only `time_off_type` membership is checked.
