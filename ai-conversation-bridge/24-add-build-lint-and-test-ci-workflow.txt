Issue Type: Feature Request
Recommended Visibility: Public issue
Suggested Title: Add CI for `ruff`, Python smoke tests, and container build verification

## Problem
The repository currently has security/scanning workflows (`CodeQL`, `Semgrep`, `TruffleHog`, `Scorecard`, Dependabot automation), but no workflow that enforces the contributor guidance in `CONTRIBUTING.md` such as `ruff check`, test execution, or container build verification.

## Proposed Solution
Add a CI workflow that runs at least:
- `ruff check`
- Python smoke tests or `pytest` once tests exist
- a minimal import/compile step
- container build verification for `chat-connector` and `mcp-demo-server`

- Add only linting without container verification
