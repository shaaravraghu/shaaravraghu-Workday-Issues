Purpose
Use this file as the filing checklist for the issue drafts in `issue-drafts/`.

Recommended process
1. Review duplicates first.
   - Open the repo issues page and search by the draft title and key nouns.
   - If a similar issue already exists, add your evidence there instead of opening a duplicate.

2. Start with the highest-signal drafts.
   - File 01, 02, 03, and 04 first.
   - Then file the documentation drift issues 09, 10, 11, and 12.
   - File the medium-risk formatter/docs UX issues afterward.

3. Use the draft contents directly.
   - Copy `ISSUE TITLE` into the GitHub issue title.
   - Copy the rest into the issue body.
   - Keep `SUGGESTED LABELS` handy for manual label selection if the repo allows it.

4. Match the repo contribution guidance.
   - Communicate on issues before starting implementation.
   - Ask whether the issue is already being worked on.
   - If you plan to fix one, comment that you want to take it.

Quality checks before submitting each issue
- Confirm the file paths and line references still match the current branch.
- Remove any statement that assumes a runtime failure unless you reproduced it.
- Keep the issue focused on one root problem.
- Add screenshots only if a docs or Storybook issue benefits from them.
- If the issue is documentation-only, say so in the title or body.

Good comment to leave after filing
"I reviewed the current repo state and attached file/line evidence. I have not opened a PR yet. Happy to help validate a fix if this issue is confirmed."
