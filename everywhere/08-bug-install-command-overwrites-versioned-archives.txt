Type: Bug Report
Title: [Bug]: `everywhere install` drops the version from the installed archive filename
Labels: bug

What happened?
The build pipeline creates `dist/<slug>-<version>.zip`, but the install command copies that archive to `<installPath>/<slug>.zip`, removing the version from the destination name.

Steps to Reproduce
1. Build or install a plugin with version `1.0.0`.
2. Bump the plugin to `1.0.1`.
3. Run `everywhere install` again to the same target directory.
4. Compare the installed filenames.

What did you expect to happen?
I expected the installed artifact name to preserve the version, or at least avoid silently overwriting the previous installed archive.

What actually happened?
The destination path is always `${slug}.zip`, so each install replaces the prior archive regardless of version.

Screenshots or Logs
Relevant lines:
- `cli/src/commands/everywhere/install.ts:57-70`
- `cli/src/build/packager.ts:37-42`

Environment Information
- Node.js version: not executed during this review
- SDK version: `0.2.2`
- Operating System: macOS
- Package manager: npm

Additional Context
If versionless overwriting is intended, the command output and docs should probably say so explicitly. Otherwise keeping the build filename convention would be less surprising.

How often does this happen?
Every time
