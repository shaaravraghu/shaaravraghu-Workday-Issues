## 🐛 Bug Report

`createStencil` appears to require unnecessary empty `modifiers: {}` objects in some extension cases, and the codebase already tracks this as a typing limitation.

## To Reproduce

1. Review `modules/styling/lib/cs.ts`, where `createStencil` includes a TODO about defaulting modifier config and fixing inference.
2. Review `modules/styling/spec/cs.spec.tsx`.
3. Note that the spec uses:
   - `modifiers: {}, // TODO: Remove this requirement`
   in base stencils before extending them with boolean modifiers.

## Expected Behavior

Base stencils should not need a placeholder empty `modifiers` object just to support later extension.

## Actual Results

The tests document an implementation or typing requirement that forces `modifiers: {}` in setup code, which adds noise to the API and suggests the generic inference is not doing what users would expect.
