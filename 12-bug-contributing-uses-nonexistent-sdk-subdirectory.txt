Type: Bug Report
Title: [Bug]: CONTRIBUTING directs contributors to `cd everywhere/sdk`, which does not match the repo layout
Labels: bug

What happened?
The setup instructions in `CONTRIBUTING.md` tell contributors to clone the repo and then run `cd everywhere/sdk`, but this repository root already contains the SDK sources and support files.

Steps to Reproduce
1. Follow the commands in `CONTRIBUTING.md`.
2. Run `git clone git@github.com:Workday/everywhere.git`.
3. Run `cd everywhere/sdk`.

What did you expect to happen?
I expected the documented path to match the actual repository layout.

What actually happened?
The instructions point at a nested `sdk` directory that does not exist in this checkout.

Screenshots or Logs
Relevant lines:
- `CONTRIBUTING.md:19-22`

Environment Information
- Node.js version: not executed during this review
- SDK version: `0.2.2`
- Operating System: macOS
- Package manager: npm

Additional Context
This is a first-run onboarding problem, so it is likely to create avoidable confusion before contributors even reach `just setup`.

How often does this happen?
Every time
